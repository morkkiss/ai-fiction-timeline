    Working Examples¶

    See accessibility done right. Each example below shows the same content twice: an inaccessible version (what most existing course pages look like) and an accessible version (the same content with the fixes applied). Both are real, working HTML — what you see is what a screen reader would see too. Below each pair is a plain-language list of what changed and why, plus copy-paste-ready source code for the accessible version.

    The interactive examples — including the equations rendered with MathML and the side-by-side iframe demos — load as a full-page tool below. If the embed doesn't load, open the standalone version in a new tab.

    Equations (MathML)¶

    Equation-heavy PDFs are the hardest accessibility problem on most courses. A picture of an equation is silent to a screen reader. MathML is structured math that the screen reader speaks aloud. The example tool above shows a working comparison.
