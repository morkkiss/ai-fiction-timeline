    Accessibility Statement¶

    This toolkit teaches the federal Title II accessibility standard. It also conforms to that standard. This page documents the conformance, explains how it was tested, and provides a contact for any accessibility issue you encounter.

    Conformance status¶

    This toolkit conforms to Web Content Accessibility Guidelines (WCAG) 2.1 Level AA — the standard required by the U.S. Department of Justice Title II Final Rule (28 CFR Part 35, Subpart H) for digital content delivered by public universities, with a compliance deadline of April 24, 2026.

    All public-facing pages of this toolkit have been tested against WCAG 2.1 Level A and Level AA criteria using automated tools and manual review.

    How conformance was verified¶

    Automated testing: Every public page (the landing page, all twelve Field Guide section pages, the Working Examples page, the Interactive Checker, and the Routing Guide) was scanned with axe-core 4.11.4, the open-source accessibility engine maintained by Deque Systems. Each scan was configured to evaluate the four standard WCAG rule sets (wcag2a, wcag2aa, wcag21a, wcag21aa).

    Result: zero violations across all scanned pages.

    Manual review: automated tools catch roughly 20–50 percent of accessibility issues. The remainder require human review. The following manual checks were performed:

    • Heading structure — every page begins with a single h1, with subsequent headings stepping down one level at a time.
    • Keyboard navigation — every interactive element (links, buttons, search box, tool filter chips, expandable details, glossary tooltips) is reachable and operable from a keyboard.
    • Skip links — every page provides a "Skip to main content" link that becomes visible on first keyboard focus.
    • Focus indicators — visible focus rings on every interactive element, retained in both light and dark modes.
    • Color contrast — text and background color pairings meet or exceed the 4.5:1 ratio required for normal text and the 3:1 ratio required for large text. The Material for MkDocs theme was selected in part because its default palette meets WCAG 2.1 AA contrast requirements without modification.
    • Iframe titles — every embedded iframe (the Working Examples demos, the Interactive Checker, the Routing Guide) carries a title attribute describing its content.
    • Form labels — every form field (search box, Checker input, alt-text suggestion field) has an explicit, programmatically associated label.
    • Language attribute — every page declares <html lang="en">.
    • Alternative text — there are no decorative or informational images in the toolkit body content. All visual elements are produced via CSS or icon fonts.

    Tooling used:

    • axe-core 4.11.4 — automated scans of all public pages
    • Material for MkDocs — theme framework with documented WCAG 2.1 AA conformance
    • Manual keyboard-only navigation testing in Safari and Chrome

    What this conformance means¶

    • Screen reader users can read the entire toolkit aloud, with proper announcement of headings, links, list structure, and interactive controls.
    • Keyboard-only users can navigate every page, every menu, every search, every interactive demo without a mouse.
    • Low-vision users can resize text up to 200% without loss of content or function, and read text at the contrast ratios required by the standard.
    • Users of dark mode or high-contrast settings see the toolkit in their preferred palette via the Material theme's automatic dark-mode toggle.
    • Mobile users see the toolkit at a layout that adapts to viewport size without horizontal scrolling.

    What this conformance does not mean¶

    This is not a third-party accessibility certification. The conformance claim above is a self-evaluation supported by automated tooling and manual review by the author. A future independent VPAT or formal accessibility audit would strengthen this claim.

    The Interactive Checker tool inside this toolkit performs accessibility checks on content the user pastes into it. Those checks are heuristic and not exhaustive — they catch common WCAG 2.1 AA violations but cannot certify any document or page as fully compliant. The toolkit's own conformance is independent of the checker's output on user-submitted content.

    Reporting an accessibility issue¶

    If any part of this toolkit is not accessible to you, please email morgan.talty@maine.edu with:

    • The URL of the page where you encountered the problem
    • A description of what was inaccessible (for example: "I'm using a screen reader and this button has no label," or "the contrast on this menu is too low to read")
    • Your assistive technology and browser, if you can include them

    Issues are addressed in the order received and corrections are pushed to this site as soon as they are tested.

    Where this conformance came from¶

    The toolkit was built by Morgan Talty (Department of English, University of Maine) as a colleague-to-colleague resource ahead of the April 24, 2026 federal Title II deadline. It is hosted on GitHub and served via raw.githack.com at no cost. It is not an official University of Maine product.
